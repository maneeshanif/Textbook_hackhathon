# 2.5 Chapter Summary

## Key Concepts

### Kinematics
- **Forward Kinematics (FK)**: Joint angles â†’ End-effector position
- **Inverse Kinematics (IK)**: Desired position â†’ Joint angles (may have multiple solutions)
- **Jacobian Matrix**: Maps joint velocities to end-effector velocities, $\dot{\mathbf{x}} = J \dot{\mathbf{q}}$
- **Singularities**: Configurations where robot loses degrees of freedom ($\det(J) = 0$)

### Dynamics
- **Equation of Motion**: $M(\mathbf{q}) \ddot{\mathbf{q}} + C(\mathbf{q}, \dot{\mathbf{q}}) \dot{\mathbf{q}} + G(\mathbf{q}) = \boldsymbol{\tau}$
- **Inverse Dynamics**: Given trajectory, compute required torques (for control)
- **Forward Dynamics**: Given torques, predict motion (for simulation)
- **Computed Torque Control**: Uses inverse dynamics to linearize the system

### Locomotion
- **Zero Moment Point (ZMP)**: Must stay inside support polygon for stability
- **Gait Phases**: Double support â†’ Single support â†’ Swing â†’ Heel strike
- **Linear Inverted Pendulum Model (LIPM)**: Simplified CoM dynamics for walking
- **Whole-Body Control**: QP-based optimization for real-time torque generation

## Technical Foundations

### Homogeneous Transformations

$$
T = \begin{bmatrix}
R & \mathbf{p} \\
0 & 1
\end{bmatrix}
$$

Combines rotation $R$ and translation $\mathbf{p}$ in a single 4Ã—4 matrix.

### Denavit-Hartenberg Parameters

Standardized method using 4 parameters per joint: $a_i, \alpha_i, d_i, \theta_i$

### ZMP Constraint

For stable walking:

$$
\mathbf{p}_{\text{ZMP}} \in \text{Support Polygon}
$$

### Capture Point

Position where robot must step to stop:

$$
\mathbf{p}_{\text{CP}} = \mathbf{p}_{\text{CoM}} + \sqrt{\frac{h}{g}} \dot{\mathbf{p}}_{\text{CoM}}
$$

## Control Strategies

### Trajectory Tracking
1. Plan desired trajectory $\mathbf{q}_d(t), \dot{\mathbf{q}}_d(t), \ddot{\mathbf{q}}_d(t)$
2. Compute torques via inverse dynamics
3. Execute with low-level motor controllers

### Balance Control
- **Ankle Strategy**: Adjust ankle torque (small disturbances)
- **Hip Strategy**: Rotate hips (medium disturbances)
- **Stepping Strategy**: Take a step (large disturbances)

### ZMP-Based Walking
1. Plan footsteps
2. Generate CoM trajectory (ZMP inside support polygon)
3. Compute joint trajectories via IK
4. Execute with inverse dynamics

- **Electric Motors**: Efficient, quiet, low maintenance

### Sensor Suite
- **Vision**: RGB cameras, depth sensors (RealSense, ZED)

- **Perception**: 10-30 Hz for vision processing

## Glossary

| Term | Definition |
|------|------------|
| **Forward Kinematics** | Mapping from joint angles to end-effector pose |
| **Inverse Kinematics** | Mapping from desired pose to joint angles |
| **Jacobian** | Matrix relating joint velocities to end-effector velocities |
| **ZMP** | Zero Moment Point - stability criterion for walking |
| **CoM** | Center of Mass |
| **LIPM** | Linear Inverted Pendulum Model - simplified walking dynamics |
| **Whole-Body Control** | Unified optimization of all joints for a task |
| **Capture Point** | Point where robot must step to stop |
